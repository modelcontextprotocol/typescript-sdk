Approach A — MVP Implementation Plan

Date: 2026-06-30  |  Status: Ready for implementation  |  Full design: APPROACH_A_TRANSPORT_LEVEL_FORK.html  |  Spec changes: transport-changes.html

Goal: Prove the routing transport concept with the smallest possible scope. A modern (2026-06) client can send a stateless tools/call to the same endpoint that serves legacy (2025-11) clients. One server, one tool registration, both protocol versions work. No MRTR, no subscriptions, no client-side transport — server-side HTTP only.

0. Background: What Changed in the MCP Spec

The MCP protocol is changing from 2025-11-25 to 2026-06-30 with breaking changes to the connection model. This section provides enough context to implement the MVP without reading the full spec diff.

0.1 Key differences

Aspect2025-11 (Legacy)2026-06 (Modern)
Connection establishmentClient sends initialize, server returns capabilities, client sends initialized. Session established.No handshake. Client may call server/discover (optional). Every request self-describes via _meta.
How server knows client capabilitiesFrom the one-time initialize exchange, stored on the connection._meta.clientCapabilities on every request.
Session identityMcp-Session-Id header. Server keeps per-session state.No session header. Stateless.
HTTP headersSingle POST endpoint, body parsed for method.Mcp-Method header (required) + MCP-Protocol-Version header (required).
Version negotiationinitialize propose/accept._meta.protocolVersion per-request. server/discover for capability discovery.
Result shapeAlways CallToolResult.result_type discriminator: "complete" / "input_required" (MRTR) / "task".

0.2 Version detection: single header check

The 2026-06 spec makes Mcp-Method a required header on every HTTP request (SEP-2243). This header did not exist in 2025-11. Its presence or absence is the definitive routing signal — no body parsing needed, no ambiguity, O(1).

function detectVersion(req: Request): 'legacy' | 'modern' {
  return req.headers.has('mcp-method') ? 'modern' : 'legacy';
}

0.3 The server/discover endpoint

The 2026-06 replacement for initialize. Servers MUST implement it. Clients MAY call it. Returns:

{
  supportedVersions: string[],  // e.g. ["2026-06-30"]
  capabilities: ServerCapabilities,
  serverInfo: { name: string, version: string },
  instructions?: string,
}

Unlike initialize, calling server/discover does not create a session or change server state.

0.4 Per-request _meta

Every 2026-06 request includes _meta in its params:

{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/call",
  "params": {
    "name": "greet",
    "arguments": { "name": "World" },
    "_meta": {
      "protocolVersion": "2026-06-30",
      "clientCapabilities": { ... },
      "clientInfo": { "name": "my-client", "version": "1.0.0" }
    }
  }
}

1. Design Principle: Swap the Transport Class

The full design (Approach A) establishes this principle: swap the transport class, everything else stays the same. McpServer, Client, tool handlers, connect() are all unchanged. The transport absorbs version detection, protocol adaptation, and session management internally.

1.1 User-facing API (MVP target)

import { McpServer } from '@modelcontextprotocol/server';
import { HTTPVersionRoutingTransport } from '@modelcontextprotocol/server';
import express from 'express';
import * as z from 'zod/v4';

// 1. McpServer (UNCHANGED)
const server = new McpServer({ name: 'greeting-server', version: '1.0.0' });

// 2. Register a tool (UNCHANGED)
server.registerTool(
  'greet',
  {
    description: 'Greet someone by name',
    inputSchema: z.object({ name: z.string() }),
  },
  async ({ name }) => ({
    content: [{ type: 'text', text: `Hello, ${name}!` }],
  }),
);

// 3. Swap transport class (ONLY CHANGE)
const transport = new HTTPVersionRoutingTransport({
  sessionIdGenerator: () => crypto.randomUUID(),
});

// 4. Connect (UNCHANGED)
await server.connect(transport);

// 5. HTTP handler
const app = express();
app.all('/mcp', async (req, res) => {
  await transport.handleRequest(req, res);
});
app.listen(3000);

1.2 What this proves

  • A 2026-06 client can POST /mcp with Mcp-Method: tools/call + _meta and get back {result_type: "complete", content: [...]}
  • A 2025-11 client can hit the same endpoint, do initialize → session → tools/call, and it still works
  • One server, one tool registration, both work
  • The registry-passing mechanism (setProtocolConfig) works

4. Implementation Phases

Phase 0: Remove TaskManager from Protocol

Prerequisite — do this first. Tasks in their current form (TaskManager, InMemoryTaskStore, InMemoryTaskMessageQueue, experimental task server/client) are being removed from the MCP spec entirely. This is aligned with MCP maintainers. The existing task implementation will not exist in any protocol version going forward.

What to remove:

  • packages/core/src/experimental/tasks/ — the entire directory
  • TaskManager references in Protocol (protocol.ts: _taskManager, processInboundRequest, routeResponse, task context in _onrequest)
  • extractTaskManagerOptions and task-related options in Server and Client constructors
  • InMemoryTaskStore, InMemoryTaskMessageQueue exports
  • Task-related test files
  • Task-related example code
  • Task capability types and schema definitions

Why first: Protocol._onrequest() currently threads every request through TaskManager.processInboundRequest(), which wraps sendNotification, sendRequest, and routeResponse. Removing this simplifies the request dispatch path that the modern protocol handler will need to understand. Building the modern handler on top of task-entangled code would mean immediately having to disentangle it.

Verification: pnpm test:all and pnpm typecheck:all must pass after removal. Any tests that exercise task functionality should be deleted, not fixed.

Phase 1: setProtocolConfig() — the registry-passing mechanism

What: Add an optional setProtocolConfig?(config: ProtocolConfig): void method to the Transport interface. Have Protocol.connect() call it if present.

// In transport.ts — extend the Transport interface:
interface Transport {
  // ... existing members ...

  /** Called during connect() to pass protocol configuration to routing transports. */
  setProtocolConfig?(config: ProtocolConfig): void;
}

interface ProtocolConfig {
  /** Live reference to Protocol._requestHandlers. */
  requestHandlers: ReadonlyMap<string, RequestHandler>;
  /** Server info (name, version). Undefined on client side. */
  serverInfo?: Implementation;
  /** Server or client capabilities. */
  capabilities?: ServerCapabilities | ClientCapabilities;
  /** Server instructions. */
  instructions?: string;
}
// In Protocol.connect() — add one line:
async connect(transport: Transport): Promise<void> {
  this._transport = transport;

  transport.onmessage = (msg, extra) => { /* existing dispatch */ };
  transport.setSupportedProtocolVersions?.(this._supportedProtocolVersions);

  // NEW: pass protocol config to routing transports
  transport.setProtocolConfig?.({
    requestHandlers: this._requestHandlers,
    serverInfo: (this as any)._serverInfo,
    capabilities: (this as any)._capabilities,
    instructions: (this as any)._instructions,
  });

  await this._transport.start();
}

Verification: Existing transports ignore setProtocolConfig (they don't implement it). All existing tests pass unchanged.

Phase 2: Modern protocol handler (stateless request dispatch)

What: A new module that handles 2026-06 requests. It reads _meta from the request, looks up the handler in the registry, executes it, and returns a response with result_type: "complete".

// Sketch: packages/server/src/server/modernHandler.ts

interface ModernHandlerOptions {
  requestHandlers: ReadonlyMap<string, RequestHandler>;
  serverInfo: Implementation;
  capabilities: ServerCapabilities;
  instructions?: string;
}

class ModernProtocolHandler {
  constructor(private options: ModernHandlerOptions) {}

  /** Handle a parsed JSON-RPC request from a 2026-06 client. */
  async handleRequest(
    request: JSONRPCRequest,
    extra?: { authInfo?: AuthInfo; request?: globalThis.Request }
  ): Promise<JSONRPCResponse> {
    const method = request.method;

    // server/discover — return capabilities
    if (method === 'server/discover') {
      return this.handleDiscover(request);
    }

    // Look up handler in the shared registry
    const handler = this.options.requestHandlers.get(method);
    if (!handler) {
      return jsonRpcError(request.id, -32601, `Method not found: ${method}`);
    }

    // Extract _meta for per-request context
    const meta = request.params?._meta;
    if (!meta?.protocolVersion) {
      return jsonRpcError(request.id, -32600, 'Missing _meta.protocolVersion');
    }

    // Build context and execute
    const ctx = this.buildContext(request, meta, extra);
    try {
      const result = await handler(request, ctx);
      return {
        jsonrpc: '2.0',
        id: request.id,
        result: { result_type: 'complete', ...result },
      };
    } catch (err) {
      return jsonRpcError(request.id, -32603, (err as Error).message);
    }
  }

  private handleDiscover(request: JSONRPCRequest): JSONRPCResponse {
    return {
      jsonrpc: '2.0',
      id: request.id,
      result: {
        supportedVersions: ['2026-06-30'],
        capabilities: this.options.capabilities,
        serverInfo: this.options.serverInfo,
        ...(this.options.instructions && { instructions: this.options.instructions }),
      },
    };
  }

  private buildContext(
    request: JSONRPCRequest,
    meta: RequestMeta,
    extra?: { authInfo?: AuthInfo; request?: globalThis.Request }
  ): ServerContext {
    // Build a ServerContext from per-request _meta
    // (same shape as legacy, different source)
    return {
      sessionId: undefined,
      mcpReq: {
        id: request.id,
        method: request.method,
        _meta: meta,
        signal: new AbortController().signal,
        send: async () => { throw new Error('Not implemented in MVP'); },
        notify: async () => { /* no-op in MVP */ },
        log: async () => { /* no-op in MVP */ },
      },
      http: extra ? { authInfo: extra.authInfo, req: extra.request } : undefined,
    };
  }
}

MVP limitations:

  • ctx.mcpReq.send() throws — no server→client requests (MRTR deferred)
  • ctx.mcpReq.log() is a no-op — in-band logging deferred
  • ctx.mcpReq.notify() is a no-op — notifications deferred
  • Only result_type: "complete" — no "input_required" or "task"

Phase 3: HTTPVersionRoutingTransport

What: The routing transport that ties everything together. Implements the Transport interface. On handleRequest(), checks the Mcp-Method header and routes to either the modern handler or a per-session legacy stack.

// Sketch: packages/server/src/server/httpVersionRoutingTransport.ts

class HTTPVersionRoutingTransport implements Transport {
  // Transport interface callbacks (set by Protocol.connect)
  onmessage?: Transport['onmessage'];
  onclose?: Transport['onclose'];
  onerror?: Transport['onerror'];

  // Protocol config (set via setProtocolConfig during connect)
  private protocolConfig?: ProtocolConfig;

  // Modern handler (created when protocolConfig is received)
  private modernHandler?: ModernProtocolHandler;

  // Legacy session map: sessionId -> {transport, server}
  private legacySessions = new Map<string, LegacySessionEntry>();

  constructor(private options: {
    sessionIdGenerator?: () => string;
  }) {}

  setProtocolConfig(config: ProtocolConfig): void {
    this.protocolConfig = config;
    this.modernHandler = new ModernProtocolHandler({
      requestHandlers: config.requestHandlers,
      serverInfo: config.serverInfo!,
      capabilities: config.capabilities as ServerCapabilities,
      instructions: config.instructions,
    });
  }

  async start(): Promise<void> {
    // Nothing to do — we handle requests on demand
  }

  async close(): Promise<void> {
    // Close all legacy sessions
    for (const [id, entry] of this.legacySessions) {
      await entry.transport.close();
      this.legacySessions.delete(id);
    }
  }

  async send(message: JSONRPCMessage): Promise<void> {
    // The routing transport does not send messages itself.
    // Each inner transport/handler sends its own responses.
    throw new Error('send() should not be called on the routing transport directly');
  }

  /** Main entry point — called by the HTTP framework handler. */
  async handleRequest(req: Request): Promise<Response> {
    // Version detection: single header check, O(1), no ambiguity
    if (req.headers.has('mcp-method')) {
      return this.handleModernRequest(req);
    } else {
      return this.handleLegacyRequest(req);
    }
  }

  private async handleModernRequest(req: Request): Promise<Response> {
    if (!this.modernHandler) {
      return jsonErrorResponse(500, 'Modern handler not initialized');
    }

    // Parse JSON-RPC body
    const body = await req.json();
    const message = JSONRPCMessageSchema.parse(body);

    if (!isJSONRPCRequest(message)) {
      return jsonErrorResponse(400, 'Expected JSON-RPC request');
    }

    // Dispatch to modern handler
    const response = await this.modernHandler.handleRequest(message, {
      authInfo: extractAuthInfo(req),
      request: req,
    });

    return new Response(JSON.stringify(response), {
      status: 200,
      headers: { 'Content-Type': 'application/json' },
    });
  }

  private async handleLegacyRequest(req: Request): Promise<Response> {
    // Route by session ID
    const sessionId = req.headers.get('mcp-session-id');

    if (sessionId) {
      const entry = this.legacySessions.get(sessionId);
      if (!entry) return jsonErrorResponse(404, 'Session not found');
      return entry.transport.handleRequest(req);
    }

    // No session — could be initialize or GET/DELETE
    if (req.method === 'POST') {
      return this.handleLegacyInitialize(req);
    }

    return jsonErrorResponse(400, 'Missing Mcp-Session-Id');
  }

  private async handleLegacyInitialize(req: Request): Promise<Response> {
    // Create a new legacy stack
    const innerTransport = new WebStandardStreamableHTTPServerTransport({
      sessionIdGenerator: this.options.sessionIdGenerator,
      onsessioninitialized: (sid) => {
        this.legacySessions.set(sid, { transport: innerTransport, server });
      },
    });

    innerTransport.onclose = () => {
      const sid = innerTransport.sessionId;
      if (sid) this.legacySessions.delete(sid);
    };

    // Create a Server connected to the inner transport
    // Server reads handlers from the shared registry
    const server = new Server(this.protocolConfig!.serverInfo!, {
      capabilities: this.protocolConfig!.capabilities as ServerCapabilities,
    });

    // Install request handlers from the shared registry
    for (const [method, handler] of this.protocolConfig!.requestHandlers) {
      server.setRequestHandler(method as any, handler as any);
    }

    await server.connect(innerTransport);
    return innerTransport.handleRequest(req);
  }
}

Phase 4: Integration test

What: A single test file that proves the routing transport works end-to-end. Registers a tool, sends both a legacy and a modern request, verifies both get the same result.

// test/httpVersionRouting.test.ts

describe('HTTPVersionRoutingTransport', () => {
  let server: McpServer;
  let transport: HTTPVersionRoutingTransport;

  beforeEach(async () => {
    server = new McpServer({ name: 'test', version: '1.0.0' });

    server.registerTool('greet', {
      inputSchema: z.object({ name: z.string() }),
    }, async ({ name }) => ({
      content: [{ type: 'text', text: `Hello, ${name}!` }],
    }));

    transport = new HTTPVersionRoutingTransport({
      sessionIdGenerator: () => randomUUID(),
    });

    await server.connect(transport);
  });

  it('handles a modern 2026-06 tools/call', async () => {
    const response = await transport.handleRequest(
      new Request('http://localhost/mcp', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'Mcp-Method': 'tools/call',
          'MCP-Protocol-Version': '2026-06-30',
        },
        body: JSON.stringify({
          jsonrpc: '2.0', id: 1,
          method: 'tools/call',
          params: {
            name: 'greet',
            arguments: { name: 'World' },
            _meta: {
              protocolVersion: '2026-06-30',
              clientCapabilities: {},
              clientInfo: { name: 'test-client', version: '1.0.0' },
            },
          },
        }),
      })
    );

    const body = await response.json();
    expect(body.result.result_type).toBe('complete');
    expect(body.result.content[0].text).toBe('Hello, World!');
  });

  it('handles a modern 2026-06 server/discover', async () => {
    const response = await transport.handleRequest(
      new Request('http://localhost/mcp', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'Mcp-Method': 'server/discover',
          'MCP-Protocol-Version': '2026-06-30',
        },
        body: JSON.stringify({
          jsonrpc: '2.0', id: 1,
          method: 'server/discover',
          params: {
            _meta: {
              protocolVersion: '2026-06-30',
              clientCapabilities: {},
              clientInfo: { name: 'test-client', version: '1.0.0' },
            },
          },
        }),
      })
    );

    const body = await response.json();
    expect(body.result.supportedVersions).toContain('2026-06-30');
    expect(body.result.serverInfo.name).toBe('test');
  });

    // Initialize
    const initResponse = await transport.handleRequest(
      new Request('http://localhost/mcp', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'Accept': 'application/json, text/event-stream',
        },
        body: JSON.stringify({
          jsonrpc: '2.0', id: 1,
          method: 'initialize',
          params: {
            protocolVersion: '2025-11-25',
            capabilities: {},
            clientInfo: { name: 'legacy-client', version: '1.0.0' },
          },
        }),
      })
    );

    const sessionId = initResponse.headers.get('mcp-session-id');
    expect(sessionId).toBeDefined();

    // ... send initialized notification, then tools/call with session ID
    // ... verify same tool result as modern path
  });

  it('same tool returns same content on both paths', async () => {
    // Call via modern path
    const modernResult = await callToolModern(transport, 'greet', { name: 'Alice' });

    // Call via legacy path (after init)
    const legacyResult = await callToolLegacy(transport, 'greet', { name: 'Alice' });

    // Content must be identical
    expect(modernResult.content).toEqual(legacyResult.content);
  });
});

6. Success Criteria

The MVP is done when:

  1. TaskManager is fully removed from Protocol/Server/Client. pnpm test:all passes (minus deleted task tests).
  2. setProtocolConfig() exists on the Transport interface. Existing transports and tests are unaffected.
  3. A modern tools/call request works — stateless, with _meta, returns {result_type: "complete"}.
  4. A modern server/discover request works — returns capabilities and server info.
  5. A legacy initialize + tools/call still works through the same transport.
  6. The integration test proves both paths return identical tool results.
  7. Existing test suite is green — no regressions in legacy behavior.
